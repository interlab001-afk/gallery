      빈도 분석 (Frequency Analysis)

      과거 50회 당첨 데이터에서 1~45번 각 번호의 출현 횟수를 집계합니다. 자주 등장한 번호는 핫 넘버, 적게 등장한 번호는 콜드 넘버로 분류하여 통계적 편향을 파악합니다.

      빈도(n) = 번호 n의 출현 횟수 / 전체 추첨 횟수
      핫 가중치: W(n) = max( freq(n), 0.5 )
      콜드 가중치: W(n) = max( maxFreq + 1 - freq(n), 0.5 )
      2

      가중 확률 선택 (Weighted Probability)

      단순 랜덤이 아닌, 각 번호에 분석 결과 기반 가중치를 부여합니다. 밸런스 모드에서는 핫 넘버 가중치 60%와 콜드 넘버 가중치 40%를 혼합하여 통계적 경향과 반전 가능성을 동시에 반영합니다.

      밸런스 가중치:
      W(n) = freq(n) × 0.6 + (maxFreq + 1 - freq(n)) × 0.4

      선택 확률: P(n) = W(n) / ΣW(i)
      3

      홀짝 균형 검증 (Odd/Even Balance)

      실제 로또 당첨 번호를 분석하면 홀수와 짝수가 균형에 가깝게 나옵니다. 생성된 번호 조합의 홀짝 비율을 실시간으로 계산하여, 극단적인 편향(예: 6:0, 0:6)을 감지하면 자동으로 재생성합니다. 이상적 비율은 3:3 또는 4:2입니다.

      허용 범위: 홀수 2~4개 (짝수 2~4개)
      검증 실패 시 → 재생성 (최대 50회 시도)
      4

      번호대 분포 최적화 (Range Distribution)

      1~10, 11~20, 21~30, 31~40, 41~45의 5개 구간에서 최소 3개 이상의 구간에 번호가 분포하도록 검증합니다. 한 구간에 번호가 과도하게 집중되는 비현실적 조합을 방지하여 실제 당첨 패턴에 가까운 결과를 생성합니다.

      구간: [1-10] [11-20] [21-30] [31-40] [41-45]
      조건: 활성 구간 ≥ 3
      검증 실패 시 → 재생성

      시스템 분석 요약

      • 과거 50회 당첨 데이터 실시간 분석
      • 3가지 전략 모드 지원 (핫/콜드/밸런스)
      • 번호별 가중치 기반 비균일 확률 분포 적용
      • 홀짝 비율 자동 균형 조정 (2:4 ~ 4:2)
      • 5개 번호 구간 중 최소 3구간 분산 보장
      • 각 세트별 홀짝 비율 및 번호대 수 실시간 표시
      • 모든 연산은 브라우저 내에서 실행 (서버 전송 없음)
      • 암호학적 보안 랜덤은 아닌 통계 기반 추천
      ⚠ 안내: 본 시스템은 과거 데이터의 통계적 패턴을 분석하여 참고용 번호를 추천합니다. 로또 추첨은 매회 독립적인 확률 사건이며, 과거 데이터가 미래 결과를 보장하지 않습니다. 본 도구는 오락 및 참고 목적이며, 당첨을 보장하지 않습니다. 책임감 있는 구매를 권장합니다.
